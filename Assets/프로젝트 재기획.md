# Trace 게임 명세서 (Game Specification)

이 문서는 "Trace" 게임의 아키텍처, 로직 및 설계를 AI가 이해하고 추가 개발을 할 수 있도록 상세히 설명합니다.

## 1. 게임 개요
**Trace**는 독특한 시간 조작 메커니즘을 사용하여 보스("Void Sentinel")와 전투를 벌이는 하이 테크 액션 게임입니다. 핵심 게임플레이는 실시간 회피와, 행동을 기록한 후 초인적인 속도로 실행하는 "TRACE Mode" 사이의 전환을 중심으로 이루어집니다.

## 2. 핵심 메커니즘

### A. 게임 상태 (`GameState`)
게임은 다음과 같은 상태를 가진 유한 상태 머신(FSM)으로 작동합니다:
- **NORMAL**: 실시간 게임플레이 상태. 플레이어는 자유롭게 이동하고 보스는 공격을 수행합니다.
- **TRACE**: `Space` 키를 누르고 있을 때 활성화됩니다. 시간이 멈추거나 매우 느려지며, 플레이어는 "고스트(Ghost)"를 조종하여 경로와 행동을 기록합니다.
- **REPLAY**: `Space` 키를 떼거나 에너지가 고갈되면 활성화됩니다. 기록된 행동들이 고속으로 재생됩니다 (전체 기록의 100%를 약 1초 내에 완료).

### B. "트레이스(Trace)" 시스템
1. **기록 (TRACE)**:
   - `TRACE` 상태 동안 게임은 고정된 비율(`FPS_TRACE = 6`)로 `TraceFrame` 객체를 기록합니다.
   - 각 프레임은 `position`(위치), `rotation`(회전), `action`(행동: MOVE 또는 ATTACK)을 저장합니다.
   - 기록은 **에너지**를 소모하며, 최대 기록 시간은 게임 시간 기준 약 4~5초입니다.
2. **실행 (REPLAY)**:
   - 저장된 `TraceFrame` 리스트를 순회합니다.
   - **보간(Interpolation)**: 고속 재생 시 부드러운 움직임을 보장하기 위해 프레임 사이의 플레이어 위치를 선형 보간(`Lerp`)합니다.
   - **액션 트리거**: 프레임에 `ATTACK` 액션이 포함된 경우, 보스와의 충돌 검사를 수행하여 데미지를 입힙니다.

## 3. 엔티티 로직 (Entity Logic)

### 플레이어 (Player)
- **이동**: WASD 키를 사용합니다.
- **체력**: 보스의 투사체에 맞으면 감소합니다.
- **에너지**: `NORMAL` 모드에서 천천히 재생되며, `TRACE` 모드에서 빠르게 소모됩니다.
- **전투**: 공격은 오직 트레이스 시스템을 통해서만 가능합니다.

### 보스 (Void Sentinel)
- **패턴**: 타이머에 따라 정해진 공격 패턴을 순환합니다.
- **투사체**: 플레이어를 향하거나 기하학적 패턴으로 움직이는 원형 투사체를 발사합니다.
- **페이즈**: 체력이 낮아질수록 패턴이 복잡해집니다 (발사 속도 증가, 투사체 수 증가 등).

## 4. 기술적 구현 세부 사항

### 렌더링 (Rendering)
- **Canvas API**: 모든 렌더링에 2D Canvas를 사용합니다.
- **시각 효과 (VFX)**:
  - **잔상(Trails)**: `REPLAY` 모드에서 고스트 잔상이 나타납니다.
  - **화면 흔들림(Screen Shake)**: 공격 성공 시 트리거됩니다.

## 5. 조작법 (Controls)
- **WASD**: 플레이어 이동(Normal) / 고스트 이동(Planning).
- **SPACE (홀드)**: TRACE 진입.
- **SPACE (릴리즈)**: REPLAY 실행.
- **마우스 클릭 (계획 모드 중)**: 현재 고스트 위치에 공격(Attack) 액션 기록.
